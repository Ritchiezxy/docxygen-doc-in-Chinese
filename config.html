格式

配置文件是自由格式的ASCII文本文件，其结构类似于Makefile，默认名称为Doxyfile。它由doxygen解析。该文件可能包含制表符和换行符，用于格式化。文件中的语句区分大小写。注释可以放在文件的任何位置(引号除外)。以两个散列字符(##)开头的注释将在更新配置文件时保留，并放在它们所处标记的前面。以配置文件开头的两个散列字符(##)开头的注释也将被保存并放在文件的开头。配置文件末尾以两个散列字符(##)开头的注释也将被保存并放在文件末尾。注释以散列字符(#)开头，以行尾结束。

该文件本质上由一个赋值语句列表组成。每个语句由用大写字母编写的TAG_NAME 、等号(=)和一个或多个值组成。如果同一标记被分配多次，则最后一次分配将覆盖任何较早的分配。对于以列表作为参数的标记，可以使用+= 操作符而不是 = 向列表添加新值。值是非空序列。如果值应该包含一个或多个空格，则必须用引号("...")包围。通过插入反斜杠(\)作为一行的最后一个字符，可以连接多行。可以使用$(ENV_VARIABLE_NAME)模式展开环境变量。

您还可以使用@INCLUDE 标签从另一个配置文件中包含配置文件的一部分，如下所示:

@INCLUDE = config_file_name

在当前工作目录中搜索包含文件。还可以指定在查找当前工作目录之前应该搜索的目录列表。通过在@INCLUDE_PATH 标签前加上 @INCLUDE 标签，例如: 

@INCLUDE_PATH = my_config_dir

与项目相关的配置选项

DOXYFILE_ENCODING

此标记指定用于随后配置文件中的所有字符的编码。默认值是UTF-8，它也是此标记第一次出现之前用于所有文本的编码。Doxygen使用libiconv (或内置到 libc中的iconv)进行代码转换。有关可能的编码列表，请参见https://www.gnu.org/software/libiconv/。

默认值是: UTF-8.

PROJECT_NAME

PROJECT_NAME 标记是一个单词(或者由双引号包围的单词序列，除非使用Doxywizard)，它应该标识为其生成文档的项目。这个名称用于大多数生成页面的标题和其他一些地方。

默认值是: My Project.

PROJECT_NUMBER

PROJECT_NUMBER 标记可用于输入项目或修订号。如果使用了某个版本控制系统，这对于归档生成的文档非常方便。

PROJECT_BRIEF

使用 PROJECT_BRIEF 标签，可以为出现在每个页面顶部的项目提供一个可选的一行描述，并且应该让查看者快速了解项目的目的。描述要简短。

PROJECT_LOGO

使用 PROJECT_LOGO 标记，可以指定文档中包含的徽标或图标。标志的最大高度不应超过55像素，最大宽度不应超过200像素。Doxygen将把徽标复制到输出目录。

OUTPUT_DIRECTORY

OUTPUT_DIRECTORY 标记用于指定生成的文档将写入的(相对或绝对)路径。如果输入了相对路径，它将相对于doxygen启动的位置。如果留空，将使用当前目录。

CREATE_SUBDIRS

如果将 CREATE_SUBDIRS 标记设置为 YES ，那么doxygen将在每种输出格式的输出目录下创建4096个子目录(分两级)，并将生成的文件分布到这些目录上。当为doxygen提供大量源文件时，启用此选项非常有用，因为将所有生成的文件放在同一个目录中会给文件系统带来性能问题。

默认值是: NO.

ALLOW_UNICODE_NAMES

如果 ALLOW_UNICODE_NAMES 标签被设置为 YES, doxygen将允许在生成的文件的名称中出现非ascii字符。如果设置为NO，非ascii字符将转义，例如_xE3_x81_x84将用于Unicode U+3044。

默认值是: NO.

OUTPUT_LANGUAGE

OUTPUT_LANGUAGE 标记用于指定编写doxygen生成的所有文档所用的语言。Doxygen将使用这些信息以适当的语言生成所有常量输出。

可能的值是: 南非荷兰语、阿拉伯语、亚美尼亚语、巴西、加泰罗尼亚人、中国人、中国传统、克罗地亚、捷克语、丹麦语、荷兰语、英语(美国),世界语、波斯语(波斯),芬兰、法国、德国、希腊、匈牙利语、印度尼西亚、意大利、日本、Japanese-en(日语与英语的消息)、韩语、Korean-en(韩语和英语消息)、拉脱维亚、立陶宛、马其顿语、挪威语、波斯(波斯语)、波兰、葡萄牙、罗马尼亚、俄罗斯、塞尔维亚、Serbian-Cyrillic、斯洛伐克、斯洛文尼亚人、西班牙语、瑞典语、土耳其人、乌克兰语和越南语。

默认值为: 英语.

OUTPUT_TEXT_DIRECTION

OUTPUT_TEXT_DIRECTION 标签用于指定编写doxygen生成的所有文档的方向。Doxygen将使用这些信息以正确的方向生成所有生成的输出。

可能的值是: None、 LTR、 RTL 和 Context。

默认值是: None.

BRIEF_MEMBER_DESC

如果将 BRIEF_MEMBER_DESC 标记设置为 YES, doxygen将在文件和类文档中列出的成员之后包含简短的成员描述(类似于Javadoc)。设置为NO 以禁用此功能。

默认值是: YES.

REPEAT_BRIEF

如果将 REPEAT_BRIEF 标记设置为 YES, doxygen将在详细描述之前预先给出成员或函数的简要描述
注意:如果将 HIDE_UNDOC_MEMBERS 和 BRIEF_MEMBER_DESC 都设置为 NO，则将完全取消简短描述。

默认值是: YES.

ABBREVIATE_BRIEF

这个标记实现了一个准智能的简短描述缩写，用于形成各种清单中的文本。如果发现此列表中的每个字符串是简要描述的主导文本，则将从文本中删除，处理整个列表后的结果将用作带注释的文本。否则，将按原样使用简要描述。如果留空，将使用以下值(将用实体的名称自动替换($name): $name类、 $name组件、 $name文件、 is、 provides、 specifies、 contains、 represents、 a、 an 以及the。

ALWAYS_DETAILED_SEC

如果 ALWAYS_DETAILED_SEC和 REPEAT_BRIEF标记都被设置为 YES ，那么doxygen将生成一个详细的部分，即使只有一个简短的描述。

默认值是: NO.

INLINE_INHERITED_MEMB

如果将 INLINE_INHERITED_MEMB标记设置为YES, doxygen将在该类的文档中显示类的所有继承成员，就好像这些成员是普通类成员一样。基类的构造函数、析构函数和赋值操作符将不显示。

默认值是: NO.

FULL_PATH_NAMES

如果 FULL_PATH_NAMES标签被设置为 YES, doxygen将在文件列表和头文件中的文件名之前添加完整的路径。如果设置为NO ，将使用使文件名惟一的最短路径

默认值是: YES.

STRIP_FROM_PATH

STRIP_FROM_PATH 标记可用于剥离用户定义的路径的一部分。只有当指定的字符串之一与路径的左侧匹配时，才会进行剥离。标记可用于显示文件列表中的相对路径。如果将运行doxygen的目录留空，则使用该目录作为要删除的路径。
注意，您可以在这里指定绝对路径，也可以指定相对路径，相对于doxygen启动的目录。

此标记要求将标记 FULL_PATH_NAMES 设置为 YES。

STRIP_FROM_INC_PATH

STRIP_FROM_INC_PATH 标记可用于剥离类文档中提到的用户定义的路径的一部分，该部分告诉读者为了使用类应该包含哪个头文件。如果留空，则只使用包含类定义的头文件的名称。否则，应该指定通常使用-I 标志传递给编译器的include路径列表。

SHORT_NAMES

如果 SHORT_NAMES 标记设置 YES, doxygen将生成更短(但可读性更差)的文件名。这很有用，因为您的文件系统不支持DOS、Mac或CD-ROM上的长名称。

默认值是: NO.

JAVADOC_AUTOBRIEF

如果 JAVADOC_AUTOBRIEF 标记被设置为 YES ，那么doxygen将把javadocstyle注释的第一行(直到第一个点)解释为简要描述。如果设置为NO, javadoc样式的行为将与常规qt样式的注释一样(因此需要显式的 @brief命令来进行简要描述)。

默认值是: NO.

JAVADOC_BANNER

如果 JAVADOC_BANNER 标记被设置为 YES ，那么doxygen将把

/***************

这样的行解释为javadocstyle注释“banner”的开头。如果设置为NO, javadoc样式的行为将与常规注释一样，并且不会被doxygen解释。

默认值是: NO.

QT_AUTOBRIEF

如果 QT_AUTOBRIEF 标记被设置为 YES ，那么doxygen将把qt样式的注释的第一行(直到第一个点)解释为简要描述。如果设置为 NO, Qt-style的行为将与普通的Qt-style注释一样(因此需要显式的\brief 命令来进行简要描述)。

默认值是: NO.

MULTILINE_CPP_IS_BRIEF

MULTILINE_CPP_IS_BRIEF 标记可以设置为 YES 处理一个多行C++特殊注释块(即//! 或 /// 注释) ，作简要说明。这曾经是默认行为。新的默认值是将多行C++注释块作为详细描述。如果您更喜欢旧的行为，则将此标记设置为YES 。
注意，将这个标签设置为 YES 也意味着rational rose注释不再被识别。

默认值是: NO.

INHERIT_DOCS

如果将INHERIT_DOCS 标记设置为 YES ，则无文档化的成员将从它重新实现的任何文档化成员那里继承文档。

默认值是: YES.

SEPARATE_MEMBER_PAGES

如果 SEPARATE_MEMBER_PAGES 标签被设置为 YES ，那么doxygen将为每个成员生成一个新页面。如果设置为NO，则成员的文档将是包含该成员的文件/类/名称空间的一部分。

默认值是: NO.

TAB_SIZE

TAB_SIZE 标签可用于设置选项卡中的空格数。Doxygen使用这个值用代码片段中的空格替换制表符。

最小值: 1， 最大值: 16， 默认值: 4。

ALIASES

此标记可用于指定作为文档中的命令的多个别名。别名的形式如下: 

 name=value

例如，添加 

 "sideeffect=@par Side Effects:\n"

将允许您在文档中放置 \sideeffect (或 @sideeffect) 命令，这将生成一个用户定义的段落，标题为“副作用:”。您可以将 \n放在别名的值部分以插入新行(在结果输出中)。您可以在别名的值部分中放入^^ 来插入换行，就像在原始文件中插入物理换行一样。当你需要一个字符{ 或 } 或 , 作为别名的一部分，你必须通过一个反斜杠 (\)，来为他们转移,这可能导致冲突的命令 \{ 或 \} 这些建议使用版本 @{ 和 @} 或者使用双转义 (\\{ 和 \\})

TCL_SUBST

此标记可用于指定多个单词-关键字映射(仅限TCL)。映射的形式是 "name=value"。例如，添加"class=itcl::class" 将允许您使用 itcl::class 含义中的命令类。

OPTIMIZE_OUTPUT_FOR_C

如果您的项目只包含C源代码，那么将 OPTIMIZE_OUTPUT_FOR_C 标记设置为YES。然后Doxygen将生成更适合C的输出。例如，使用的一些名称将有所不同。所有成员的名单将被删除，等等。

默认值是: NO.

OPTIMIZE_OUTPUT_JAVA

如果您的项目只包含Java或Python源代码，那么将 OPTIMIZE_OUTPUT_JAVA 标记设置为 YES 。然后Doxygen将生成更适合该语言的输出。例如，名称空间将以包的形式显示，限定范围将看起来不同，等等。

默认值是: NO.

OPTIMIZE_FOR_FORTRAN

如果您的项目包含Fortran源代码，则将 OPTIMIZE_FOR_FORTRAN 标记设置为 YES 。然后Doxygen将生成为Fortran量身定制的输出。

默认值是: NO.

OPTIMIZE_OUTPUT_VHDL

如果项目包含VHDL源代码，则将OPTIMIZE_OUTPUT_VHDL 标记设置为 YES 。然后Doxygen将生成为VHDL量身定制的输出。

默认值是: NO.

OPTIMIZE_OUTPUT_SLICE

 FILE_VERSION_FILTER = "/bin/sh versionfilter.sh"

#!/bin/sh
cvs status $1 | sed -n 's/^[ \]*Working revision:[ \t]*\([0-9][0-9\.]*\).*/\1/p'


Subversion的shell脚本示例: 

#!/bin/sh
svn stat -v $1 | sed -n 's/^[ A-Z?\*|!]\{1,15\}/r/;s/ \{1,15\}/\/r/;s/ .*//p'


ClearCase的例子过滤器: 

FILE_VERSION_FILTER = "cleartool desc -fmt \%Vn"

此标记可用于指定doxygen解析的源文件的字符编码。doxygen内部使用UTF-8编码。Doxygen使用libiconv (或内置到libc中的 iconv )进行代码转换。有关可能的编码列表，请参阅 libiconv文档 。

默认值是: UTF-8.

FILE_PATTERNS

如果 INPUT 标记的值包含目录，可以使用 FILE_PATTERNS 标记指定一个或多个通配符模式(如 *.cpp 和 *.h) 来过滤目录中的源文件。
注意, 对于自定义扩展或不直接支持的扩展，还需要为扩展设置EXTENSION_MAPPING ，否则doxygen不会读取文件。
如果留白，测试以下模式:*.c、 *.cc、 *.cxx、 *.cpp、 *.c++, *.java、 *.ii、 *.ixx、 *.ipp、 *.i++, *.inl、 *.idl、 *.ddl、 *.odl、 *.h、 *.hh、 *.hxx、 *.hpp、 *.h++, *.cs、 *.d、 *.php、 *.php4、 *.php5、 *.phtml、 *.inc、 *.m、 *.markdown、 *.md、 *.mm、 *.dox、 *.py、 *.pyw、 *.f90、 *.f95、 *.f03、 *.f08、 *.f、 *.for、 *.tcl、 *.vhd、 *.vhdl、 *.ucf、 *.qsf and *.ice.

RECURSIVE

RECURSIVE 标记可用于指定是否也应该搜索输入文件的子目录。

默认值是: NO.

EXCLUDE

EXCLUDE 标记可用于指定应该从INPUT 源文件中排除的文件和/或目录。通过这种方式，您可以轻松地从目录树中排除子目录，该目录树的根由 INPUT 标记指定。
注意，相对路径相对于运行doxygen的目录。

EXCLUDE_SYMLINKS

可以使用EXCLUDE_SYMLINKS 标记来选择是否将符号链接(Unix文件系统特性)的文件或目录排除在输入之外。

默认值是: NO.

EXCLUDE_PATTERNS

如果 INPUT 标记的值包含目录，则可以使用 EXCLUDE_PATTERNS 标记指定一个或多个通配符模式，以从这些目录中排除某些文件。
注意 通配符是针对具有绝对路径的文件进行匹配的，因此要排除所有测试目录，例如使用模式 */test/*

EXCLUDE_SYMBOLS

可以使用EXCLUDE_SYMBOLS 标记指定一个或多个应该从输出中排除的符号名称(名称空间、类、函数等)。符号名可以是一个完全限定的名称、一个单词，或者如果使用通配符 * ，可以是一个子字符串。 例子:ANamespace、 AClass、 AClass::ANamespace、 ANamespace::*Test
注意，通配符是针对具有绝对路径的文件进行匹配的，因此要排除所有测试目录，请使用模式 */test/*

EXAMPLE_PATH

EXAMPLE_PATH标记可用于指定包含示例代码片段的一个或多个文件或目录(请参阅\include 命令)。

EXAMPLE_PATTERNS

如果EXAMPLE_PATH 标记的值包含目录，则可以使用 EXAMPLE_PATTERNS 标记指定一个或多个通配符模式(如 *.cpp 和 *.h) 来过滤目录中的源文件。如果留空，则包含所有文件。

EXAMPLE_RECURSIVE

如果 EXAMPLE_RECURSIVE 标记被设置为 YES ，那么将搜索子目录，寻找与 \include 或 \dontinclude 命令一起使用的输入文件，而不考虑RECURSIVE 标记的值。

默认值是: NO.

IMAGE_PATH

IMAGE_PATH标记可用于指定一个或多个文件或目录，其中包含要包含在文档中的图像(请参阅\image命令)。

INPUT_FILTER

INPUT_FILTER 标记可用于指定doxygen应该调用的程序，以过滤每个输入文件。Doxygen将通过执行(通过popen())命令来调用过滤器程序:
<filter> <input-file>
其中<filter> 是 INPUT_FILTER 标签的值，<input-file> 是输入文件的名称。然后Doxygen将使用过滤器程序写入标准输出的输出。如果指定了 FILTER_PATTERNS ，这个标记将被忽略。
注意,过滤器不能添加或删除行;它在扫描代码之前应用，而不是在生成输出代码时应用。如果添加或删除行，锚将不能正确放置。
注意 对于自定义扩展或不直接支持扩展您还需要设置扩展EXTENSION_MAPPING 否则文件不能被doxygen适当处理。

FILTER_PATTERNS

FILTER_PATTERNS 标记可用于根据每个文件模式指定过滤器。Doxygen将比较每个模式的文件名，如果匹配，则应用过滤器。过滤器是表单的列表:(例如 *.cpp=my_cpp_filter)。有关如何使用过滤器的更多信息，请参见INPUT_FILTER 。如果FILTER_PATTERNS 标记为空，或者没有匹配文件名的模式，则应用INPUT_FILTER 。
注意,对于自定义扩展或不直接支持扩展您还需要设置扩展 EXTENSION_MAPPING 否则文件不能被doxygen适当处理

FILTER_SOURCE_FILES

如果 FILTER_SOURCE_FILES 标记被设置为 YES,输入滤波器(如果设置使用INPUT_FILTER)也将被用来过滤输入文件用于生产源文件浏览(即当SOURCE_BROWSER 设置为 YES)。

默认值是: NO.

FILTER_SOURCE_PATTERNS

FILTER_SOURCE_PATTERNS 标记可用于为每个文件模式指定源过滤器。模式将覆盖FILTER_PATTERN 的设置(如果有的话)，还可以使用*.ext= 禁用特定模式的源过滤(因此无需命名过滤器)。

这个标签需要标记 FILTER_SOURCE_FILES 设置为 YES。

USE_MDFILE_AS_MAINPAGE

如果 USE_MDFILE_AS_MAINPAGE 标记引用作为输入一部分的标记文件的名称，那么它的内容将放在主页 (index.html)上。如果您有一个项目(例如GitHub)，并且希望重用介绍页面以获得doxygen输出，那么这将非常有用。

与源文件浏览相关的配置选项

SOURCE_BROWSER

如果 SOURCE_BROWSER 标记被设置为 YES 然后将生成的源文件的列表。文档化的实体将与这些源相互引用。
注意: :摆脱所有的源代码生成的输出,也确保 VERBATIM_HEADERS 设置为 NO。

默认值是: NO.

INLINE_SOURCES

将 INLINE_SOURCES 标记设置为 YES 将直接将函数、类和枚举的主体包含到文档中。

默认值是: NO.

STRIP_CODE_COMMENTS

将 STRIP_CODE_COMMENTS 标记设置为 YES 将指示doxygen对生成的源代码片段隐藏任何特殊的注释块。普通的C、C++和Fortran注释将始终保持可见。

默认值是: YES.

REFERENCED_BY_RELATION

如果 REFERENCED_BY_RELATION 标记被设置为 YES ,然后对于每个被文档化的实体，所有引用它的被文档化的函数将被列出。

默认值是: NO.

REFERENCES_RELATION

如果 REFERENCES_RELATION 标记被设置为 YES ，然后对于每一个文档化的函数，所有被文档化的被该函数调用/使用实体将被列出。

默认值是: NO.

REFERENCES_LINK_SOURCE

如果REFERENCES_LINK_SOURCE 标记被设置为 YES 并且 SOURCE_BROWSER 标记被设置为 YES 那么 REFERENCES_RELATION 和 REFERENCED_BY_RELATION 列表功能的超链接将链接到源代码。否则它们将链接到文档。

默认值是: YES.

SOURCE_TOOLTIPS

如果启用了 SOURCE_TOOLTIPS (缺省值)，那么在源代码中悬停一个超链接，将显示一个工具提示，其中包含一些附加信息，如原型、简要描述以及指向定义和文档的链接。由于这会使HTML文件变大，加载大文件的速度变慢，所以可以选择禁用此特性。

默认值是: YES.

这个标签需要把标记 SOURCE_BROWSER 设置为 YES。

USE_HTAGS

如果 USE_HTAGS 标记被设置为 YES ,然后源代码的引用将指向由htags(1)工具生成的HTML, 而不是doxygen浏览器内置的源。htags 工具是GNU全球源标记系统的一部分(参见 https://www.gnu.org/software/global/global.html)。 您将需要4.8.6或更高版本。
要使用它做以下工作:

  1. 安装最新版本的 global
  2. 在配置文件中启用 SOURCE_BROWSER 和 USE_HTAGS 。
  3. 确保INPUT 指向源代码树的根
  4. 正常运行doxygen
    Doxygen将调用 htags (这将反过来调用 gtags)，因此这些工具必须在命令行(即搜索路径)中可用。
    结果:现在指向源代码的链接将指向htags的输出，而不是doxygen生成的源浏览器。

默认值是: NO.

这个标签需要把标记 SOURCE_BROWSER 设置为 YES。

VERBATIM_HEADERS

如果将 VERBATIM_HEADERS 标记设置为 YES ，那么doxygen将为指定包含的每个类生成头文件的逐字副本。设置为 NO 以禁用此功能。

参见
Section \class。

默认值是: YES.

CLANG_ASSISTED_PARSING

如果 CLANG_ASSISTED_PARSING 标记被设置为 YES 那么doxygen clang 解析器 将使用更精确的解析以降低开销。这对于模板丰富的C++代码尤其有用，因为doxygen的内置解析器缺乏必要的类型信息。

注意
此选项的可用性取决于是否使用CMake的-Duse_libclang=ON 选项生成doxygen。

默认值是: NO。

CLANG_OPTIONS

如果启用了clang辅助解析，则可以向编译器提供调用编译器时通常使用的命令行选项。注意,包括路径已经被设置为指定的文件和目录doxygen INPUT 和 INCLUDE_PATH。

这个标签需要把标记 CLANG_ASSISTED_PARSING 设置为 YES。

CLANG_DATABASE_PATH

如果启用了clang辅助解析，您可以为clang解析器提供文件构建时使用的编译数据库 的路径。这相当于为clang工具指定“-p”选项，例如clang-check。然后将这些选项传递给解析器。

注意
此选项的可用性取决于是否使用CMake的 -Duse_libclang=ON 选项生成doxygen。

ALPHABETICAL_INDEX

如果 ALPHABETICAL_INDEX 标记被设置为 YES将生成所有组合的的字母索引。如果项目包含大量类、结构、联合或接口，则启用此功能。

COLS_IN_ALPHA_INDEX

COLS_IN_ALPHA_INDEX 标记可用于指定按字母顺序拆分索引列表的列数。

IGNORE_PREFIX

如果一个项目中的所有类都以一个公共前缀开始，那么所有类都将放在按字母顺序排列的索引的相同标题下。 IGNORE_PREFIX 标记可用于指定在生成索引头时应该忽略的前缀(或前缀列表)。

这个标签需要把标记 ALPHABETICAL_INDEX 设置为 YES。

与HTML输出相关的配置选项

GENERATE_HTML

如果 GENERATE_HTML 标记被设置为 YES,doxygen将生成HTML输出。

默认值是: YES.

HTML_OUTPUT

HTML_OUTPUT标签用于指定HTML文档将放在何处。如果输入了一个相对路径，OUTPUT_DIRECTORY 的值将放在它的前面。

默认目录是:html.

这个标签需要把标记 GENERATE_HTML 设置为 YES。

HTML_FILE_EXTENSION

HTML_FILE_EXTENSION 标记可用于为每个生成的HTML页面指定文件扩展名(例如: .htm, .php, .asp)。

默认值是: .html.

HTML_HEADER

的默认头开始，然后修改文件 new_header.html。

有关如何生成Doxygen通常使用的默认头文件的信息，请参见 Doxygen 用法 一节。

注意
头文件可能会发生更改，因此在升级到新版本的doxygen时，通常必须重新生成默认的头文件。下面的标记在页眉和页脚中有特殊的含义:
$title
将被替换为页面的标题。
$datetime
将替换为当前日期和时间。
$date
将用当前日期替换。
$year
将以当前年份替换。
$doxygenversion
会被doxygen版本取代吗
$projectname
将替换为项目的名称(请参见 PROJECT_NAME)
$projectnumber
将替换为项目编号(请参见 PROJECT_NUMBER)
$projectbrief
将替换为项目简要描述(请参见 PROJECT_BRIEF)
$projectlogo
将替换为项目标识(参见 PROJECT_LOGO)
$treeview

$search

$mathjax
将替换为指向MathJax特性所需的javascript和样式表的链接(或禁用USE_MATHJAX 时的空字符串)。
$relpath^
如果启用 CREATE_SUBDIRS ，命令 $relpath^ 可以用来生成HTML输出目录根目录的相对路径，例如使用 $relpath^doxygen.css来引用标准样式表。

 <!--END BLOCKNAME-->

DISABLE_INDEX

GENERATE_TREEVIEW
当启用 GENERATE_TREEVIEW 选项时，将该块中的内容复制到输出。
SEARCHENGINE
当启用 SEARCHENGINE 选项时，此块中的内容将复制到输出中。
PROJECT_NAME
当 PROJECT_NAME选项不为空时，将块内的内容复制到输出。
PROJECT_NUMBER
当 PROJECT_NUMBER 选项不为空时，块内的内容将复制到输出。
PROJECT_BRIEF
当 PROJECT_BRIEF 选项不为空时，块内的内容将复制到输出。
PROJECT_LOGO
当 PROJECT_LOGO 选项不为空时，块内的内容将复制到输出。
TITLEAREA
当每个页面顶部都有一个标题时，此块中的内容将复制到输出中。如果填写了PROJECT_NAME、 PROJECT_BRIEF和 PROJECT_LOGO ，或者启用了 DISABLE_INDEX 和 SEARCHENGINE ，就会出现这种情况。

这个标签需要把标记GENERATE_HTML 设置为 YES。

HTML_FOOTER

HTML_FOOTER 标签可用于为每个生成的HTML页面指定用户定义的HTML页脚。如果标记为空，doxygen将生成一个标准页脚。

有关如何生成默认页脚以及可以在页脚中使用哪些特殊命令的更多信息，请参见 HTML_HEADER 一节。

有关如何生成Doxygen通常使用的默认页脚的信息，请参见Doxygen 用法 for information on how to generate the default footer that doxygen normally uses.

这个标签需要把标记 GENERATE_HTML 设置为 YES。

HTML_STYLESHEET

HTML_STYLESHEET 标记可用于指定每个HTML页面使用的用户定义的层叠样式表。它可以用来微调HTML输出的外观。如果左空白doxygen将生成一个默认样式表。

有关如何生成Doxygen通常使用的样式表的信息，请参见 Doxygen 用法 一节。

注意
建议使用HTML_EXTRA_STYLESHEET 代替这个标记，因为它更健壮，而且这个标记 (HTML_STYLESHEET) 将来会过时。

这个标签需要把标记 GENERATE_HTML 设置为 YES。

HTML_EXTRA_STYLESHEET
HTML_EXTRA_STYLESHEET 标记可用于指定其他用户定义的层叠样式表，这些样式表包含在doxygen创建的标准样式表之后。使用此选项可以否决某些样式方面。这比使用HTML_STYLESHEET 更可取，因为它不替换标准样式表，因此对未来的更新更健壮。Doxygen将样式表文件复制到输出目录。
注意
额外样式表文件的顺序很重要(例如，列表中的最后一个样式表覆盖了列表中前一个样式表的设置)。下面是一个例子样式表，它给内容区域一个固定的宽度: 
body {
        background-color: #CCC;
        color: black;
        margin: 0;
}

div.contents {
        margin-bottom: 10px;
        padding: 12px;
        margin-left: auto;
        margin-right: auto;
        width: 960px;
        background-color: white;
        border-radius: 8px;
}

#titlearea {
        background-color: white;
}

hr.footer {
        display: none;
}

.footer {
        background-color: #AAA;
}
这个标签需要把标记 GENERATE_HTML 设置为 YES。
HTML_EXTRA_FILES

HTML_EXTRA_FILES 标记可用于指定一个或多个额外的图像或其他源文件，这些文件应该复制到HTML输出目录中。注意,这些文件将被复制到基本HTML输出目录。在HTML_HEADER和/或HTML_FOOTER文件中使用 $relpath^ 标记来加载这些文件。在HTML_STYLESHEET 文件中，只使用文件名。还请注意，文件将按原样复制;没有可用的命令或标记。

这个标签需要把标记 GENERATE_HTML 设置为 YES。

HTML_COLORSTYLE_HUE

HTML_COLORSTYLE_HUE 标记控制HTML输出的颜色。Doxygen将根据这种颜色调整样式表和背景图像中的颜色。Hue被指定为色轮上的一个角度，有关更多信息，请参见https://en.wikipedia.org/wiki/Hue 例如，值0表示红色，60表示黄色，120表示绿色，180表示青色，240表示蓝色，300表示紫色，360表示红色。

最小值： 0, 最大值： 359, 默认值： 220.

这个标签需要把标记 GENERATE_HTML 设置为 YES。

HTML_COLORSTYLE_SAT

HTML_COLORSTYLE_SAT 标记控制HTML输出中颜色的纯度(或饱和度)。对于值0，输出将只使用灰度。255的值将产生最鲜艳的颜色。

最小值： 0, 最大值： 255, 默认值： 100.

这个标签需要把标记 GENERATE_HTML 设置为 YES。

HTML_COLORSTYLE_GAMMA

HTML_COLORSTYLE_GAMMA HTML_COLORSTYLE_GAMMA标记控制应用于HTML输出中颜色亮度组件的gamma校正。低于100的值逐渐使输出变亮，而高于100的值则使输出变暗。值除以100是实际应用的，所以80代表0.8的值，220代表2.2的值，100不改变。

最小值： 40, 最大值： 240, 默认值： 80.

这个标签需要把标记 GENERATE_HTML 设置为 YES。

HTML_TIMESTAMP

如果 HTML_TIMESTAMP 标记被设置为 YES, 那么每个生成的HTML页面的页脚包含生成页面时的日期和时间。将此设置为 YES可以帮助显示doxygen最后一次运行的时间以及文档是否更新。

默认值是: NO.

这个标签需要把标记 GENERATE_HTML 设置为 YES。

HTML_DYNAMIC_MENUS

如果 HTML_DYNAMIC_MENUS 标记被设置为 YES 那么生成的HTML文档将包含一个主索引与垂直导航菜单,通过Javascript动态创建。如果禁用，导航索引将由静态嵌入每个HTML页面的多个级别的选项卡组成。禁用此选项以支持没有Javascript的浏览器，类似于Qt帮助浏览器。

默认值是: YES.

这个标签需要把标记 GENERATE_HTML 设置为 YES。

HTML_DYNAMIC_SECTIONS

如果 HTML_DYNAMIC_SECTIONS 标记被设置为 YES 标记被设置为是的然后生成的HTML文档将包含几部分,各部分可隐藏和显示在页面加载之后。

默认值是: NO.

这个标签需要把标记 GENERATE_HTML 设置为 YES。

HTML_INDEX_NUM_ENTRIES

使用HTML_INDEX_NUM_ENTRIES 可以控制各种树结构索引中初始显示的首选项数;用户稍后可以动态地展开和折叠条目。Doxygen将树扩展到最多可以看到指定数目的条目的级别(除非完全折叠的树已经超过这个数量)。因此，默认情况下，设置条目1的数量将生成一个完整的折叠树。0是一个表示无限个条目的特殊值，默认情况下将生成一个完整的展开树。

最小值： 0, 最大值： 9999, 默认值： 100.

这个标签需要把标记 GENERATE_HTML 设置为 YES。

GENERATE_DOCSET

如果 GENERATE_DOCSET 标记被设置为 YES,将生成额外的索引文件作为输入,可以用于 苹果的Xcode 3集成开发环境,介绍了OSX 10.5(Leopard)。 要创建文档集，doxygen将在HTML输出目录中生成Makefile。运行 make 将在该目录中生成docset，运行make install 将在 ~/Library/Developer/Shared/Documentation/DocSets 中安装docset，以便Xcode在启动时找到它。更多信息请参见 https://developer.apple.com/library/archive/featuredarticles/DoxygenXcode/_index.html 。

默认值是: NO.

Suppose you have a simple project consisting of two files: a source file example.cc and a header file example.h。 Then a minimal configuration file is as simple as: 

INPUT            = example.cc example.h

Assuming the example makes use of Qt classes and perl is located in /usr/bin, a more realistic configuration file would be: 

PROJECT_NAME     = Example
INPUT            = example.cc example.h
WARNINGS         = YES
TAGFILES         = qt.tag
SEARCHENGINE     = NO

To generate the documentation for the QdbtTabular package I have used the following configuration file: 

PROJECT_NAME     = QdbtTabular
OUTPUT_DIRECTORY = html
WARNINGS         = YES
INPUT            = examples/examples.doc src
FILE_PATTERNS    = *.cc *.h
INCLUDE_PATH     = examples
TAGFILES         = qt.tag
SEARCHENGINE     = YES

To regenerate the Qt-1.44 documentation from the sources, you could use the following configuration file: 

PROJECT_NAME         = Qt
OUTPUT_DIRECTORY     = qt_docs
HIDE_UNDOC_MEMBERS   = YES
HIDE_UNDOC_CLASSES   = YES
ENABLE_PREPROCESSING = YES
MACRO_EXPANSION      = YES
EXPAND_ONLY_PREDEF   = YES
SEARCH_INCLUDES      = YES
FULL_PATH_NAMES      = YES
STRIP_FROM_PATH      = $(QTDIR)/
PREDEFINED           = USE_TEMPLATECLASS Q_EXPORT= \
                       QArrayT:=QArray \
                       QListT:=QList \
                       QDictT:=QDict \
                       QQueueT:=QQueue \
                       QVectorT:=QVector \
                       QPtrDictT:=QPtrDict \
                       QIntDictT:=QIntDict \
                       QStackT:=QStack \
                       QDictIteratorT:=QDictIterator \
                       QListIteratorT:=QListIterator \
                       QCacheT:=QCache \
                       QCacheIteratorT:=QCacheIterator \
                       QIntCacheT:=QIntCache \
                       QIntCacheIteratorT:=QIntCacheIterator \
                       QIntDictIteratorT:=QIntDictIterator \
                       QPtrDictIteratorT:=QPtrDictIterator
INPUT                = $(QTDIR)/doc \
                       $(QTDIR)/src/widgets \
                       $(QTDIR)/src/kernel \
                       $(QTDIR)/src/dialogs \
                       $(QTDIR)/src/tools
FILE_PATTERNS        = *.cpp *.h q*.doc
INCLUDE_PATH         = $(QTDIR)/include
RECURSIVE            = YES

For the Qt-2.1 sources I recommend to use the following settings: 

PROJECT_NAME          = Qt
PROJECT_NUMBER        = 2.1
