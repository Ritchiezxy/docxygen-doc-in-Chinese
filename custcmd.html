Doxygen提供了大量的特殊命令、 XML命令以及HTML命令。可用于增强或构造注释块内的文档。如果出于某种原因需要定义新命令，可以通过别名定义来实现。

别名的定义应该在配置文件中使用ALIASES 配置标记指定。

简单的别名

别名最简单的形式是对该形式的简单替换

 name=value

例如，定义以下别名: 

 ALIASES += sideeffect="\par Side Effects:\n"

将允许您将命令\sideeffect (或@sideeffect)放在文档中，这将导致一个用户定义的带有标题副作用的段落:

注意，您可以将\\n放在别名的值部分来插入新行(在结果输出中)。您可以在别名的值部分中放入^^ 来插入换行，就像在原始文件中插入物理换行一样。

注意当你需要一个别名值的一部分的字符{ 或 } 或 , 您必须通过反斜杠(\)来转义它们，这可能会导致与命令\{ 和 \} 冲突，因此建议使用版本 @{ 和 @} 或使用双转义 (\\{ 和 \\})

还请注意，如果您愿意，您可以重新定义现有的特殊命令。

一些命令，比如 \xrefitem，被设计成与别名结合使用。

带参数的别名

别名也可以有一个或多个参数。在别名定义中，然后需要指定大括号之间的参数数量。在定义的值部分，您可以放置\x 标记，其中 “x” 表示以1开头的参数号。

下面是一个别名定义的例子，只有一个参数: 

ALIASES += l{1}="\ref \1"

在注释块中，您可以按如下方式使用它

/** 查看 \l{SomeClass} 获取更多信息。 */

这与如下书写是一样的 

/** 查看 \ref SomeClass 获取更多信息。 */

注意，您可以通过带有多个参数的版本重载别名，例如: 

ALIASES += l{1}="\ref \1"
ALIASES += l{2}="\ref \1 \"\2\""

注意，别名定义中的引号必须用反斜杠转义。

使用这些别名定义，我们可以在注释块中编写 

/** 查看 \l{SomeClass,Some Text} 获取更多信息 */

，它将扩展到 

/** 查看 \ref SomeClass "Some Text" 获取更多信息 */

，其中带有单个参数的命令仍然可以像前面所示那样工作。

别名也可以用其他别名表示，例如，一个新的命令\reminder 可以通过一个中间的 \xreflist 命令表示为\xrefitem，如下所示: 

ALIASES += xreflist{3}="\xrefitem \1 \"\2\" \"\3\" "
ALIASES += reminder="\xreflist{reminders,Reminder,Reminders}"

请注意，如果使用逗号作为分隔符来分隔具有多个参数的别名，如果要在命令中放置逗号，则需要使用反斜杠转义它，即

\l{SomeClass,Some text\, with an escaped comma}

在上面的例子中给出了\l 的别名定义。

嵌套自定义命令

您可以将命令用作别名的参数，包括使用别名定义的命令。

作为一个例子，考虑以下别名定义

ALIASES += Bold{1}="<b>\1</b>"
ALIASES += Emph{1}="<em>\1</em>"

Inside a comment block you can now use: 

/** This is a \Bold{bold \Emph{and} Emphasized} text fragment. */

which will expand to 

/** This is a <b>bold <em>and</em> Emphasized</b> text fragment. */
