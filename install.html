在UNIX上从源代码编译

如果下载了源发行版，至少需要以下步骤来构建可执行文件：

  • GNU flex、 bison、 libiconv 、GNU make以及strip
  • 你需要python （ 2.6及以上版本， 请浏览 https://www.python.org）。
  • 为了为您的平台生成MakefileMakefile，您需要cmake 的3.1.3或更高版本。

为了充分利用doxygen的特性，应该安装以下附加工具。

  • Qt软件的GUI工具包，Qt 4.3或更高版本(包括Qt 5)。这是构建GUI前端doxywizard所需要的。
  • 发行版:例如TeX Live 这是生成、Postscript和PDF输出所需要的。
  • 包含依赖关系图、图形继承图和协作图所需的图形可视化工具包1.8.10或更高版本。果您自己编译graphviz，请确保包含freetype支持(这需要freetype库和头文件)，否则图形将不会呈现正确的文本标签。
  • 对于HTML输出中的公式(当不使用MathJax时)或不希望使用pdflatexpdflatex时，需要ghostscript解释器。你可以在www.ghostscript.com上找到它。

编译工作现按以下步骤进行:

  1. 解压归档文件，除非你已经如下操作了: 

    gunzip doxygen-$VERSION.src.tar.gz    # uncompress the archive
tar xf doxygen-$VERSION.src.tar       # unpack it

  2. 创建一个构建目录(例如在源文件树中)

    cd doxygen-$VERSION
mkdir build
cd build

  3. 运行makefile生成器cmake

    cmake -G "Unix Makefiles" ..

    cmake尝试确定您使用的平台，并寻找所需的工具。如果缺少什么，它会报告。

    如果您已经安装了Qt-4.3或更高版本，并希望构建GUI前端，您应该启用它，如下所示:

    cmake -Dbuild_wizard=YES ..

    有关使用其他配置选项的概述

    cmake -L ..

  4. 运行make编译程序: 

    make

    程序编译时应该没有问题，并且会在build目录中的bin目录生成一个可用的二进制文件(doxygen 和可选生成的doxywizard）。

  5. 可选:生成用户手册。

    cmake -Dbuild_doc=YES ..
make docs

    让doxygen生成HTML和PDF文档。

    构建目录中的HTML目录现在将包含HTML文档(只需将HTML浏览器指向HTML目录中的index.html文件)。

  6. 可选:静态链接

    如果你想构建一个静态链接版本的doxygen嵌入libclang，你需要首先需要使用以下选项构建LLVM和clang的源文件:

    cmake -DLIBCLANG_BUILD_STATIC=ON \
      -DBUILD_SHARED_LIBS=OFF \
      -DLLVM_ENABLE_PIC=OFF \
      -DLLVM_BUILD_LLVM_DYLIB=OFF \
      -DLLVM_BUILD_LLVM_C_DYLIB=OFF \
      -DLLVM_ENABLE_TERMINFO=OFF \
      path_to_llvm_root_source_dir

    然后用这些选项构建doxygen: 

    cmake -DCMAKE_BUILD_TYPE=Release \
      "-DCMAKE_FIND_LIBRARY_SUFFIXES=.a" \
      "-ldl;-lz;-lpthread" \
      -Duse_libclang=YES \
      path_to_doxygen_root_source_dir

在UNIX上安装二进制文件

编译完源代码后做一个make install来安装doxygen。如果下载了UNIX的二进制发行版，请键入:

./configure
make install

二进制文件安装在目录<prefix>/bin。使用make install_docs 将文档和示例安装到<docdir>/doxygen。

<prefix> 默认为/usr/local ，但是可以使用configure脚本的--prefix 选项进行更改。默认的<docdir> 目录是<prefix>/share/doc/packages ，可以使用configure脚本的--docdir 选项进行更改。

另外，您还可以手动地将二进制文件从bin directory manually to some bin 目录复制到搜索路径中的某个目录。这足以使用doxygen。

注意
为此，您需要GNU安装工具(它是coreutils包的一部分)。其他安装工具可能会将二进制文件放在错误的目录中!

如果您有RPM或DEP包，那么请遵循这些包所需的标准安装过程。

在Windows上从源代码编译

从1.8.10版本开始，构建文件需要由cmake生成。cmake可以从https://cmake.org/download/下载。

目前只测试了Visual Studio 2013的express版本，但是其他版本应该也可以工作。

或者，您可以使用Cygwin或MinGW以UNIX方式编译doxygen。

下一步是安装bison和flex的现代版本(参见https://sourceforge.net/projects/winflexbison/）。安装并将它们添加到path后，将win_flex.exe 重命名为 flex.exe ，将 win_bison.exe 重命名为 bison.exe) 此外，还必须安装python (版本2.6或更高，请参见https://www.python.org)。编译过程中需要这些包。

下载doxygen的源代码tarball并将其放在某个地方(例如，使用c:\tools)

现在启动visual studio原生命令shell(适用于x86或x64)并键入

cd c:\tools
tar zxvf doxygen-x.y.z.src.tar.gz

要解压缩源代码(可以从http://gnuwin32.sourceforge.net/packages.html获得tar)。或者，您可以使用一个解包程序，比如7-Zip(参见https://www.7-zip.org/)，或者使用现代Windows系统的内置解包功能。

现在，您的环境已经设置为生成doxygen所需的项目文件。

cd到doxygen-x.y.z文件所在的目录，创建并cd到build目录。

mkdir build
cd build
cmake -G "Visual Studio 12 2013" ..

注意编译Doxywizard需要Qt 4.3或更新版本(请参见https://www.qt.io/developers/)。

还要阅读下一节，了解运行doxygen时可能需要安装的附加工具，以启用某些特性。

在Windows上安装二进制文件

Doxygen是一个自安装的归档文件，所以安装非常简单。跟着对话就行了。

安装之后，还建议下载并安装GraphViz(强烈建议使用2.20或更高版本)。Doxygen可以使用GraphViz包的dot 工具来呈现更好的图表，请参阅配置文件中的 HAVE_DOT 选项。

如果您想在配置文件中生成压缩的HTML文件(请参阅GENERATE_HTMLHELP)，那么您需要Microsoft HTML帮助中心，你可以从下载。

如果您想在配置文件中生成Qt压缩的帮助文件(参见QHG_LOCATION)，那么您需要qhelpgenerator，它是Qt的一部分，您可以从Qt软件下载下载Qt。

为了生成PDF输出或使用科学公式，还需要安装LaTeX 和 Ghostscript。

对于 存在许多发行版。doxygen的常用工具是MikTex 和 proTeXt。

Ghostscriptt可以从downloaded下载。

安装 和 Ghostscript 之后，需要确保工具latex.exe、 pdflatex.exe、 和 gswin32c.exe 出现在命令框的搜索路径中。如果不确定，请遵循以下 说明，并从命令框中运行命令来验证它是否工作。
