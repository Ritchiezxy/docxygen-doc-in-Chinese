Doxygen
组织

Doxygen有三种机制把东西组合在一起。一种机制在全局级别工作，为每个组创建一个新页面。这些组在文档中称为“模块”。第二种机制在某个复合实体的成员列表中工作，称为 “成员组”。对于 页面 ，还有第三种分组机制称为子分页。

模块

模块是在一个单独的页面上将内容组合在一起的一种方法。您可以将一组文档作为一个整体，以及所有单独的成员。组的成员可以是文件、名称空间、类、函数、变量、枚举、typedefs和定义，也可以是其他组。

要定义一个组，您应该将\defgroup 命令放在一个特殊的注释块中。该命令的第一个参数是一个标签，它应该惟一地标识组。
第二个参数是组的名称或标题，因为它应该出现在文档中。

您可以通过在其文档块中放置\ingroup命令使实体成为特定组的成员。

为了避免将 \ingroup 命令放在每个成员的文档中，您还可以通过组前的开放标记@{和组后的关闭标记@} 将成员分组在一起。标记可以放在组定义的文档中，也可以放在单独的文档块中。

组本身也可以使用这些分组标记嵌套。

当您不止一次使用相同的组标签时，您将得到一个错误消息。如果不希望doxygen强制使用唯一的标签，那么可以使用\addtogroup 而不是 \defgroup。它可以像\defgroup一样使用，但是当已经定义了组之后，它会悄悄地将现有文档与新文档合并在一起。对于这个命令，组的标题是可选的，因此可以使用

/** \addtogroup <label> 
 *  @{
 */
...

/** @}*/

向在其他地方详细定义的组添加其他成员。

注意复合实体(如类、文件和名称空间)可以放入多个组,但成员(如变量、函数、类型定义和枚举)只能在一个组的成员(这个限制是为了避免模棱两可的链接目标情况下成员不是记录在它的类中,名称空间或文件,但只作为组的一部分可见)。

Doxygen将把成员放入定义具有最高“优先级”的组中:例如，显式的\ingroup通过@{ @}覆盖隐式分组定义。具有相同优先级的冲突分组定义将触发警告，除非其中一个定义针对没有任何显式文档的成员。

下面的例子将VarInA放入组A中，并通过将其放入组IntVariables中来悄悄地解决IntegerVariable的冲突，因为IntegerVariable的第二个实例没有文档说明:

/**
 * \ingroup A
 */
extern int VarInA;

/**
 * \defgroup IntVariables 全局整型变量
 * @{
 */

/** 一个整型变量 */
extern int IntegerVariable;

/**@}*/

....

/**
 * \defgroup Variables 全局变量
 */
/**@{*/

/** 在组A中的一个变量 */
int VarInA;

int IntegerVariable;

/**@}*/

分组定义的优先级是(从高到低):\ingroup、 \defgroup、 \addtogroup、 \weakgroup。\weakgroup 命令与 \addtogroup一样，具有较低的优先级。添加它是为了允许“惰性”分组定义:您可以在.h文件中使用具有更高优先级的命令来定义.c文件中的层次结构和\weakgroup ，而不必完全复制层次结构。

例如：
/** @defgroup group1 第1组
* 这是第1组
* @{
*/
/** @brief 组1中的类C1 */
class C1 {};
/** @brief class 组1中的类C2 */
class C2 {};
/** 组1中的函数 */
void func() {}
/** @} */ // 组1末尾
/**
* @defgroup group2 第2组
* 这是第2组
*/
/** @defgroup group3 第3组
* 这是第3组
*/
/** @defgroup group4 第3组
* @ingroup group3
* 组4是组3的子组
*/
/**
* @ingroup group2
* @brief 组2中的类C3
*/
class C3 {};
/** @ingroup group2
* @brief 组2中的类C4
*/
class C4 {};
/** @ingroup group3
* @brief 类C5在@link group3 中的组@endlink中。
*/
class C5 {};
/** @ingroup group1 group2 group3 group4
* 命名空间N1在第4组中
* @sa @link group1 第1组@endlink, group2, group3, group4
*
* 请参阅 @ref mypage2
*/
namespace N1 {};
/** @file
* @ingroup group3
* @brief 该文件在组3中
*/
/** @defgroup group5 第5组
* 这是第5组
* @{
*/
/** @page mypage1 这是第5组中的1个段落
* 第1段的文本
*/
/** @page mypage2 这是第5组中的另外一段
* 这是第2端的文本
*/
/** @} */ // 组5末尾
/** @addtogroup group1
*
* 第1组的更多文档注释
* @{
*/
/** 组1中的另外一个函数 */
void func2() {}
/** 组1的另一个函数 */
void func3() {}
/** @} */ // 组1末尾

成员组

如果一个化合物(例如一个类或文件)有很多成员，通常需要将它们组合在一起。Doxygen已经根据类型和保护级别自动将内容分组，但是您可能觉得这还不够，或者默认的分组是错误的。例如，因为您认为不同(语法)类型的成员属于同一个(语义)组。

成员组由

///@{ 
  ...
///@}

块或 

/**@{*/ 
  ... 
/**@}*/

块定义(如果您喜欢C风格的注释)。注意，组的成员应该在成员组的身体内部。

在块的开始标记之前，可以放置一个单独的注释块。这个块应该包含@name (或 \name)命令，用于指定组的头。注释块还可以包含关于组的更详细的信息。

不允许嵌套成员组。

如果所有成员组的成员内部类有相同的类型和保护级别(例如所有静态公共成员),那么整个成员组将被显示为类型/保护层级的子组(组显示为一个分段的例如“静态公共成员”一节)。如果两个或多个成员具有不同的类型，则将该组放在与自动生成的组相同的级别。如果希望强制类的所有成员组位于顶层，则应该在类的文档中放置\nosubgrouping 命令。

例如：
/** 一个类。细节 */
class Memgrp_Test
{
public:
///@{
/** 两个成员的文档相同。细节 */
void func1InGroup1();
void func2InGroup1();
///@}
/** 不属于任何组的函数。细节. */
void ungroupedFunction();
void func1InGroup2();
protected:
void func2InGroup2();
};
void Memgrp_Test::func1InGroup1() {}
void Memgrp_Test::func2InGroup1() {}
/** @name Group2
* 组2的描述
*/
///@{
/** 组2中的函数2。 细节。 */
void Memgrp_Test::func2InGroup2() {}
/** 组2中的函数1。细节 */
void Memgrp_Test::func1InGroup2() {}
///@}
/*! \file
* 该文件的若干文档
*/
//!@{
//! 此组所有成员的一个描述
//! (因为DISTRIBUTE_GROUP_DOC在配置文件中是YES)
#define A 1
#define B 2
void glob_func();
//!@}

在这里，Group1显示为“Public Members”的一个小节。Group2是一个单独的部分，因为它包含了具有不同保护级别的成员(即public和protected)。

子页面

可以使用\page 和 \mainpage 命令将信息分组到页面中。通常，这会导致一个平面的页面列表，其中“main”页面是列表中的第一个页面。

使用\subpage命令向页面添加额外的结构通常比使用模块小节 中描述的方法添加结构更自然和方便。

对于页面A，\subpage 命令添加一个页面链接到另一个页面,同时使页面B a这页的子的影响使两组GA和GB, GB是遗传算法的一部分,页面页面放在组GA, B组GB。
