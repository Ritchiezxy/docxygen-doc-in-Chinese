Doxygen 用法

Doxygen是一个基于命令行的实用工具。在命令行中使用 doxygen 选项调用 --help ，可以得到该程序用法的简要说明。

所有选项都包含一个前导字符 “-”，后跟一个字符和一个或多个参数(取决于选项)。

要为您的项目生成手册，您通常需要遵循以下步骤:

  1. 您可以使用特殊的文档块来记录源代码(请参阅“特殊注释块”一节)。
  2. 通过使用-g选项调用doxygen生成一个配置文件(请参阅配置一节): 
    doxygen -g <config_file>
  3. 编辑配置文件，使其与项目匹配。在配置文件中，可以指定输入文件和许多可选信息。
  4. 让doxygen根据配置文件中的设置生成文档: 
    doxygen <config_file>

如果您有一个用旧版本的doxygen生成的配置文件，您可以通过运行带有-u选项的doxygen将其升级到当前版本。

doxygen -u <config_file>
原始配置文件中的所有配置设置都将复制到新配置文件中。任何新选项都有其默认值。注意，您可能在原始配置文件中添加的注释将丢失。

微调的输出

如果你想微调输出的方式，doxygen允许你生成默认的样式表，标题，和脚注文件，你可以编辑后:

  • 对于HTML输出，可以使用以下命令生成默认的头文件(见 HTML_HEADER)、默认的页脚(见 HTML_FOOTER)和默认的样式表(见HTML_STYLESHEET):

    doxygen -w html header.html footer.html stylesheet.css <config_file>

    config_file是可选的。当省略config_file时，doxygen将搜索一个名为Doxyfile的文件并处理它。当这个也没有被发现时，它将使用默认设置。

  • 对于 输出，您可以使用以下命令生成refman.tex 的第一部分和最后一部分(参见 LATEX_HEADER 和 LATEX_FOOTER)，以及该header包含的样式表(通常是doxygen.sty): 
    doxygen -w latex header.tex footer.tex doxygen.sty <config_file>
  • 如果您需要非默认选项(例如使用额外的包)，您需要使配置文件正确设置这些选项，然后在生成的文件之后指定配置文件(首先备份配置文件，以免忘记指定某个输出文件)。For RTF output, you can generate the default style sheet file (see RTF_STYLESHEET_FILE) using: 
    doxygen -w rtf rtfstyle.cfg
警告
使用自定义头文件时，您需要负责适当包含doxygen所需的脚本和样式表，这取决于配置选项，在升级到新的doxygen版本时可能会发生变化。
注意
  • 如果不希望为配置文件中的每个项提供文档，则可以使用可选的-s选项。这可以与-u选项结合使用，以添加或删除现有配置文件中的文档。如果您将配置文件作为bug报告的一部分发送给我，请使用-s选项!
  • 要使doxygen读/写为标准输入/输出，而不是从/到文件，使用"-"作为文件名。
