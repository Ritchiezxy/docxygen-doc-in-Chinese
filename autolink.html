大多数文档系统都有特殊的‘参见’ 部分，可以在其中插入到其他文档片段的链接。虽然doxygen也有一个命令来启动这样一个节(请参阅\sa小节），但是它允许您将这种链接放在文档的任何地方。对于 文档，将编写对页码的引用，而不是链接。此外，文档末尾的索引可用于快速查找成员、类、名称空间或文件的文档。对于手册页，不生成参考信息。

下一节将展示如何生成指向源文件中各种文档化实体的链接。

链接到网页和邮件地址

Doxygen将自动用链接(HTML)替换文档中发现的任何URL和邮件地址。要手动指定链接文本，请使用HTML“a” 标签。

<a href="linkURL">链接文本</a>

它将被doxygen自动转换成其他输出格式。

链接到类

文档中与已文档化的类相对应并包含至少一个非小写字符的所有单词将自动替换为指向包含该类文档的页面的链接。如果您想防止与文档化的类相对应的单词被链接替换，那么您应该在单词前面加上%。要链接到全小写符号，请使用\ref。

链接到文件

所有包含点(.)而不是单词中最后一个字符的单词都被认为是文件名。如果该单词确实是文档化输入文件的名称，则会自动创建到该文件文档的链接。

链接到函数

如果遇到下列模式之一，就会创建到函数的链接:

  1. <functionName>"("<argument-list>")"
  2. <functionName>"()"
  3. "::"<functionName>
  4. (<className>"::")n<functionName>"("<argument-list>")"
  5. (<className>"::")n<functionName>"("<argument-list>")"<modifiers>
  6. (<className>"::")n<functionName>"()"
  7. (<className>"::")n<functionName>

这里 n>0.

注意 1:
函数参数应该用正确的类型指定，即“fun(const std::string&，bool)”或“()”来匹配任何原型。
注意 2:
成员函数修饰符(如“const”和“volatile”)用于标识目标，即“func(int) const”和“func(int)”针对不同的成员函数。
注意 3:
对于Javadoc兼容性，在上面的模式中可以使用#而不是a::。
注意 4:
在包含成员foo的类的文档中，对全局变量的引用使用“::foo”，而#foo将链接到成员。

对于非重载成员，可以省略参数列表。

如果函数被重载，并且没有指定匹配的参数列表(即使用模式2或模式6)，则会创建到其中一个重载成员的文档的链接。

对于成员函数，可以省略类范围(如模式4到模式7中使用的), 如果：

  1. 模式指向与包含该模式的文档块属于同一类的文档化成员。
  2. 与包含模式的文档块对应的类有一个基类，该类包含与模式匹配的文档化成员。

链接到其他成员

所有这些实体都可以以与前一节描述的相同的方式链接到。为了清晰起见，建议在本例中只使用模式3和模式7。

例子：
/*! \file autolink.cpp
测试自动链接生成。
指向Autolink_Test类成员的链接: Autolink_Test::member,
更具体的链接到每个重载成员:
Autolink_Test::member(int)和Autolink_Test#member(int,int)
指向Autolink_Test的受保护成员变量的链接: Autolink_Test#var,
指向全局枚举类型#GlobEnum的链接。
指向已定义#ABS(x)的链接。
链接到Autolink_Test类的析构函数: Autolink_Test::~Autolink_Test,
到typedef:: B的链接。
指向枚举类型Autolink_Test::EType的链接
链接到一些枚举值Autolink_Test::Val1和::GVal2
*/
/*!
因为这个文档块属于Autolink_Test类，所以不会生成到
Autolink_Test的链接。
链接到构造函数的两种方法是: #Autolink_Test 和 Autolink_Test().
到析构函数的链接如下: #~Autolink_Test 和 ~Autolink_Test().
链接到这个类中的一个成员: member().
更具体的链接到每个重载成员:
member(int) 和 member(int,int).
指向变量#var的链接。
一个到全局typedef::B的链接。
指向全局枚举类型#GlobEnum的链接。
到已定义ABS(x)的链接。
一个指向变量\link #var的链接，使用另一个文本\endlink作为链接。
指向枚举类型#EType的链接。
一些枚举值的链接: \link Autolink_Test::Val1 Val1 \endlink 和 ::GVal1.
最后但并非最不重要的一个文件链接: autolink.cpp.
\sa 在see also部分中，任何单词都被选中，因此
EType、Val1、GVal1、~Autolink_Test和member将被HTML中的链接替换。
*/
class Autolink_Test
{
public:
Autolink_Test(); //!< constructor
~Autolink_Test(); //!< destructor
void member(int); /**< A member function. Details. */
void member(int,int); /**< An overloaded member function. Details */
/** An enum type. More details */
enum EType {
Val1, /**< enum value 1 */
Val2 /**< enum value 2 */
};
protected:
int var; /**< A member variable */
};
/*! details. */
Autolink_Test::Autolink_Test() { }
/*! details. */
Autolink_Test::~Autolink_Test() { }
/*! A global variable. */
int globVar;
/*! A global enum. */
enum GlobEnum {
GVal1, /*!< global enum value 1 */
GVal2 /*!< global enum value 2 */
};
/*!
* A macro definition.
*/
#define ABS(x) (((x)>0)?(x):-(x))
typedef Autolink_Test B;
/*! \fn typedef Autolink_Test B
* A type definition.
*/

typedefs

包含类、结构和联合的Typedefs，比如

typedef struct StructName TypeName

为StructName创建别名一样，当遇到StructName本身或TypeName时，将生成指向StructName的链接。

例子：
/*! \file restypedef.cpp
* 解析typedefs的一个例子
*/
/*! \struct CoordStruct
* 一个坐标。
*/
struct CoordStruct
{
/*! x坐标 */
float x;
/*! y坐标 */
float y;
};
/*! 为CoordStruct创建类型名称 */
typedef CoordStruct Coord;
/*!
* 此函数返回\a c1和\a c2的加法，即:
* (c1.x+c2.x,c1.y+c2.y)
*/
Coord add(Coord c1,Coord c2)
{
}
