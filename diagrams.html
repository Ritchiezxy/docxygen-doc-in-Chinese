Doxygen
图形和图表

Doxygen内置了对生成C++类继承图的支持。

Doxygen可以使用graphviz中的“dot”工具来生成更高级的图表。Graphviz是一个开源的、跨平台的图形绘制工具包，可以在http://www.graphviz.org/ 找到它

如果环境变量的path中有“dot”工具的路径，可以在配置文件中将 HAVE_DOT 设置为 YES ，让doxygen使用它。

Doxygen使用“dot”工具生成以下图形:

  • 将绘制类层次结构的图形表示和文本表示。目前这个特性只支持HTML。
    警告:当您有一个非常大的类层次结构，其中许多类派生自一个公共基类时，生成的图像可能会变得太大，以至于某些浏览器无法处理。
  • 将为每个文档化的类生成一个继承图，显示直接和间接继承关系。这将禁止生成内置的类继承关系图。
  • 对于包含至少一个其他文件的每个文档文件，将生成包含依赖关系图。这个特性目前只支持HTML和RTF。
  • 还生成了一个逆包含依赖关系图，显示一个(标头)文件，其他文件包括它。
  • 为每个文档化的类和结构绘制一个图表，将显示:
    • 与基类的继承关系。
    • 与其他结构和类的使用关系(例如，类A有一个成员变量m_a，类型为B，然后A有一个指向B的箭头，以m_a作为标签)。
  • 如果 CALL_GRAPH 被设置为 YES, ，则会为每个函数绘制图形化的调用图，显示该函数直接或间接调用的函数(请参阅\callgraph和\hidecallgraph)。
  • 如果CALLER_GRAPH 被设置为 YES,，则会为每个函数绘制图形化的调用者图形，显示直接或间接调用该函数的函数(请参阅 \callergraph 部分和 \hidecallergraph部分).

使用布局文件可以确定实际显示了哪些图形。

选项 DOT_GRAPH_MAX_NODES 和 MAX_DOT_GRAPH_DEPTH 可用于限制各种图形的大小。

HTML和RTF中的类图元素有以下含义:

  • 黄色框表示一个类。方框的右下角可以有一个小标记，以指示类包含隐藏的基类。对于类图，当前树的最大宽度是8个元素。如果树较宽，则会隐藏一些节点。如果框中填充了虚线模式，则继承关系是虚的。
  • 白框表示类的文档当前已显示。
  • 灰色框表示未归档的类。
  • 纯深蓝色箭头表示公共继承。
  • 带虚线的深绿色箭头 表示受保护的继承。
  • 点状的深绿色箭头 表示私有继承。

中的类图元素有以下含义:

  • 白方框表示一个类。框的右下角有一个标记，表示该类有隐藏的基类。如果框中有虚线边框，则表示虚拟继承。
  • 实线箭头表示公共继承。
  • 虚线箭头表示受保护的继承。
  • 点箭头表示私有继承。

dot工具生成的图形元素有以下含义:

  • 白框表示类、结构或文件。
  • 带有红色边框的框表示节点的箭头比显示的多!换句话说:该图相对于此节点被截断。图有时被截断的原因是为了防止图像变得太大。对于用dot doxygen生成的图形，尝试将结果图像的宽度限制在1024像素。
  • 一个黑框表示类的文档为当前显示。
  • 深蓝色箭头表示包含关系(对于包含依赖关系图)或公共继承(对于其他图)。
  • 深绿色箭头表示受保护的继承。
  • 暗红色箭头表示私有继承。
  • 紫色虚线箭头表示“使用”关系，箭头的边缘用负责关系的变量标记。类A使用类B，如果类A有一个C类型的成员变量m，其中B是C的子类型(例如C可以是B, B*， T<B>*等)。

下面是一些头文件，它们一起显示了doxygen可以生成的各种图表:

diagrams_a.h

#ifndef _DIAGRAMS_A_H
#define _DIAGRAMS_A_H
class A { public: A *m_self; };
#endif

diagrams_b.h

#ifndef _DIAGRAMS_B_H
#define _DIAGRAMS_B_H
class A;
class B { public: A *m_a; };
#endif

diagrams_c.h

#ifndef _DIAGRAMS_C_H
#define _DIAGRAMS_C_H
#include "diagrams_c.h"
class D;
class C : public A { public: D *m_d; };
#endif

diagrams_d.h

#ifndef _DIAGRAM_D_H
#define _DIAGRAM_D_H
#include "diagrams_a.h"
#include "diagrams_b.h"
class C;
class D : virtual protected A, private B { public: C m_c; };
#endif

diagrams_e.h

#ifndef _DIAGRAM_E_H
#define _DIAGRAM_E_H
#include "diagrams_d.h"
class E : public D {};
#endif
