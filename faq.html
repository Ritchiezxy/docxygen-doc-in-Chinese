如何在HTML的索引页上获取信息?

您应该在注释块中使用\mainpage命令，如下所示: 

/*! \mainpage 我的个人索引页
 *
 * \section intro_sec 介绍
 *
 * 这是引言。
 *
 * \section install_sec 安装
 *
 * \subsection step1 Step 1: 打开这个盒子
 *  
 * 等等……
 */

帮助，我的类/文件/名称空间的一些/所有成员为什么没有文档化?

检查以下几点:

  1. 您的类/文件/名称空间有文档化吗?如果没有，则不会从源中提取它，除非配置文件中将 EXTRACT_ALL 设置为 YES 。
  2. 会员是私人的吗?如果是，则必须将EXTRACT_PRIVATE 设置为 YES ，以使它们出现在文档中。

  3. 您的类中是否有一个函数宏不以分号结束(例如MY_MACRO())?如果是这样，你必须指示doxygen的预处理程序删除它。

    这通常归结为配置文件中的以下设置:

    ENABLE_PREPROCESSING   = YES
MACRO_EXPANSION        = YES
EXPAND_ONLY_PREDEF     = YES
PREDEFINED             = MY_MACRO()=

    有关更多信息，请阅读手册的预处理部分 。

当我将EXTRACT_ALL设置为NO时，文档中没有显示我的函数。

为了将全局函数、变量、枚举、typedefs和定义记录下来，您应该使用包含\file (或 @file) 命令的注释块记录这些命令所在的文件。

或者，您可以使用 \ingroup 命令将所有成员放到一个组(或模块)中，然后使用包含\defgroup 命令的注释块为这个组编写文档。

对于成员函数或属于名称空间一部分的函数，您应该记录类或名称空间。

带有自定义扩展名的文件不再(正确)解析。

Doxygen只解析指定为输入的文件(通过INPUT 标记)和匹配指定扩展名的文件(在FILE_PATTERNS中提到)，然后通过排除列出为 EXCLUDE 的文件或匹配 EXCLUDE_PATTERNS设置的模式的文件来减少文件列表。

在过去，doxygen将所有扩展名未知的文件解析为C文件，这可能导致不希望的结果。从1.8.8版开始，doxygen就要求您指定一个映射，用于指定要使用哪个解析器的特定文件扩展名。这个映射是使用 EXTENSION_MAPPING 标签指定的。如果没有指定映射，文件的内容将被忽略。

如何让doxygen忽略一些代码片段?

新的、最简单的方法是在代码的开头添加一个带有\cond 命令的注释块，在代码的末尾添加一个带有\endcond 命令的注释块，这些注释块应该被忽略。当然，这应该在同一个文件中。

但你也可以使用doxygen的预处理器:如果你把

#ifndef DOXYGEN_SHOULD_SKIP_THIS

 /* 必须被doxygen跳过的代码 */

#endif /* DOXYGEN_SHOULD_SKIP_THIS */

在该隐藏和放置的方块周围: 

  PREDEFINED = DOXYGEN_SHOULD_SKIP_THIS

在配置文件中，只要 ENABLE_PREPROCESSING 设置为 YES就应该跳过所有块。

如何更改类文档中#include之后的内容?

在大多数情况下，可以使用STRIP_FROM_INC_PATH 剥离用户定义的路径部分。

您还可以按照以下方式文档注释类

/*! \class MyClassName include.h path/include.h
 *
 *  为MyClassName添加注释
 */

要使doxygen将

#include <path/include.h>

放入类MyClassName的文档中，而不考虑包含MyClassName定义的实际头文件的名称。

如果你想要doxygen显示包含文件应该使用引号而不是尖括号，你应该输入: 

/*! \class MyClassName myhdr.h "path/myhdr.h"
 *
 *  为MyClassName添加注释
 */

如何结合使用标签文件与压缩HTML?

如果要从一个压缩的HTML文件a.chm 引用到另一个名 b.chm的压缩HTML文件，a.chm中的链接必须具有以下格式: 

<a href="b.chm::/file.html">

不幸的是，只有当两个压缩的HTML文件位于同一个目录中时，这种方法才有效。

因此，您必须将为所有项目生成的 index.chm 文件重命名为惟一的文件，并将所有 .chm 文件放在一个目录中。

假设有一个项目a使用标记文件b.tag 引用一个项目 b那么您可以将项目a的index.chm 重命名为a.chm ，将项目b的 index.chm 重命名为 b.chm。在项目a的配置文件中，你可以写: 

TAGFILES = b.tag=b.chm::

我不喜欢放在每个HTML页面上面的快速索引，我该怎么办?

您可以通过将DISABLE_INDEX 设置为 YES来禁用索引。然后，您可以通过编写自己的头文件并将其提供给HTML_HEADER来放入自己的头文件中。

整个HTML输出看起来不同，而我只想使用我自己的HTML头文件

您可能忘记包含doxygen生成的样式表doxygen.css 。您可以通过将 

<LINK HREF="doxygen.css" REL="stylesheet" TYPE="text/css">

放在HTML页面的头部部分来包含它。

为什么doxygen使用Qt?

最重要的原因是通过QFile、QFileInfo、QDir、QDate、QTime和QIODevice类为大多数Unices和Windows提供了一个平台抽象。另一个原因是良好和错误免费的实用程序类，如QList, QDict, QString, QArray, QTextStream, QRegExp, QXML等。

如何从目录树中排除所有测试目录?

只需在配置文件中放入这样一个排除模式:

EXCLUDE_PATTERNS = */test/*

Doxygen会自动在运行的文本中生成一个指向类MyClass的链接。我该如何在某个地方预防呢?

在类名前面放一个%。比如:% MyClass。然后Doxygen将删除%并保持单词不链接。

我最喜欢的编程语言是X.我还能使用doxygen吗?

不，不是这样;doxygen需要理解它所读取内容的结构。如果你不介意花点时间，这里有几个选择:

  • 如果X的语法接近C或C++，那么稍微调整src/scanner.l 以支持该语言可能并不难。这对于doxygen直接支持的所有其他语言(例如Java、IDL、C#、PHP)都是这样做的。
  • 如果X的语法有些不同，那么可以编写一个输入过滤器，将X转换成类似于C/C++的东西，以便doxygen能够理解(这种方法适用于VB、对象Pascal和Javascript，请参见http://www.doxygen.org/download.html#helpers)。
  • 如果语法完全不同，可以为X编写一个解析器，并编写一个后端，生成与src/scanner.l (以及src/tagreader.cpp)在读取标记文件时生成的语法树类似)类似的语法树。

帮助!我得到了一个神秘的消息“输入缓冲区溢出，不能扩大缓冲区，因为扫描仪使用拒绝”

当doxygen的词汇扫描器有一条规则一次性匹配超过256K的输入字符时，就会发生此错误。我在一个非常大的生成文件(>256K 行)上看到过这种情况，其中内置的预处理器将其转换为一个空文件(使用>256K的新行)。另一种可能发生这种情况的情况是，如果代码中的行超过256K个字符。

如果您遇到这种情况并希望我修复它，您应该向我发送一个触发消息的代码片段。要解决这个问题，可以在文件中放入一些换行符，将其分割成更小的部分，或者使用exclude将其从输入中排除。

解决这个问题的另一种方法是使用cmake命令的选项:

-Denlarge_lex_buffers=<size>

其中<size> 是输入(YY_BUF_SIZE) 和读取 (YY_READ_BUF_SIZE) 缓冲区的新大小。如果这个选项没有给出默认值262144(即256K)，那么将使用这个选项。

在latex目录中运行make时，我得到“TeX容量超出”。现在怎么办呢?

您可以编辑texmf.cfg文件来增加各种缓冲区的默认值，然后运行“texconfig init”。

为什么通过STL类的依赖关系没有显示在点图中?

Doxygen不知道STL类，除非打开选项BUILTIN_STL_SUPPORT。

我可以从命令行配置doxygen吗?

不是通过命令行选项，但是doxygen可以从stdin读取数据，所以您可以通过管道将数据传输到stdin。下面是一个如何从命令行覆盖配置文件中的选项的例子(假设是类似UNIX的环境):

( cat Doxyfile ; echo "PROJECT_NUMBER=1.0" ) | doxygen -

对于Windows，下面的代码也可以做到同样的效果:

( type Doxyfile & echo PROJECT_NUMBER=1.0 ) | doxygen.exe -

如果指定多个具有相同名称的选项，则doxygen将使用最后一个选项。要附加到现有选项，可以使用+=操作符。

doxygen这个名字是怎么来的?

Doxygen的名字来自于对单词document和generator的处理。

documentation -> docs -> dox
generator -> gen

当时，我正在研究lex和yacc，其中很多东西都以“yy”开头，所以“y”就溜了进来，让东西可以发音了(正确的发音是doc -ee-gen，所以有一个长“e”)。

开发doxygen的原因是什么?

我曾经编写过一个基于Qt库的GUI小部件(它仍然可以在 https://sourceforge.net/projects/qdbttabular/ 上找到，但自2002年以来就没有更新过)。Qt已经很好地生成了文档(使用的是他们不想发布的内部工具)，我手工编写了类似的文档。这是一个噩梦般的维护，所以我想要一个类似的工具。我看了一下Doc++，但那还不够好(它不支持信号和插槽，也没有我逐渐喜欢上的Qt外观和感觉)，所以我开始编写自己的工具……
