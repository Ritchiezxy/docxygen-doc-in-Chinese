可执行的doxygen是解析源文件并生成文档的主程序。有关更详细的使用信息，请参见Doxygen 用法。

可以选择使用可执行的doxywizard，这是一个图形化前端 ，用于编辑doxygen使用的配置文件，并在图形化环境中运行doxygen。对于Mac OS X doxywizard将通过单击doxygen应用程序图标启动。

步骤 0: 检查doxygen是否支持您的编程语言

首先，确保您的编程语言有合理的机会被doxygen识别。默认情况下支持这些语言:C、C++、C#、Objective-C、IDL、Java、VHDL、PHP、Python、Tcl、Fortran和D。可以配置特定的文件类型扩展来使用特定的解析器:有关详细信息，请参阅配置/扩展映射外，使用预处理器程序可以支持完全不同的语言:有关详细信息，请参阅帮助者页面。

步骤 1:创建配置文件

Doxygen使用一个配置文件来确定它的所有设置。每个项目都应该有自己的配置文件。项目可以由单个源文件组成，也可以是递归扫描的整个源文件树。

为了简化配置文件的创建，doxygen可以为您创建一个模板配置文件。要从命令行调用doxygen，可以使用-g选项: 

doxygen -g <config-file>

其中<config-file>是配置文件的名称。如果省略文件名，将创建一个名为 Doxyfile的文件。如果名称为<config-file>的文件已经存在，doxygen将在生成配置模板之前将其重命名为<config-file>.bak 。如果您使用- (即负号)作为文件名，那么doxygen将尝试从标准输入(stdin)读取配置文件，这对于脚本编写非常有用。

配置文件的格式类似于(简单的)Makefile。它由表单的若干赋值(标签)组成:

TAGNAME = VALUE or
TAGNAME = VALUE1 VALUE2 ...

您可能可以将生成的模板配置文件中的大多数标记的值保留为默认值。有关配置文件的详细信息，请参阅配置 一节。

如果您不希望使用文本编辑器编辑配置文件，您应该看看doxywizard，它是一个GUI前端，可以创建、读取和写入doxygen配置文件，并允许通过对话框输入配置选项来设置配置选项。

对于由几个C和/或C++源文件和头文件组成的小型项目，可以将INPUT标记保留为空，doxygen将在当前目录中搜索源文件。

如果您有一个包含源目录或树的较大项目，您应该将根目录或目录分配给INPUT 标记，并将一个或多个文件模式添加到FILE_PATTERNS标记(例*.cpp *.h）。只有匹配其中一种模式的文件才会被解析(如果省略模式，doxygen支持的文件类型将使用典型模式列表)。对于源树的递归解析，必须将RECURSIVE 设置为 YES。要进一步微调解析的文件列表，可以使用EXCLUDE 和 EXCLUDE_PATTERNS 标记。例如，要省略源树中的所有 test 目录，可以使用:

EXCLUDE_PATTERNS = */test/*

如果您开始为一个现有的项目使用doxygen(因此没有doxygen知道的任何文档)，您仍然可以了解结构是什么以及文档化的结果是什么样子的。为此，您必须将配置文件中的EXTRACT_ALL标记设置为YES。然后，doxygen将假装您的源文件中的所有内容都被记录了下来。请注意，因此，只要将EXTRACT_ALL 设置为 YES，就不会生成关于非法成员的警告。

要分析一个现有的软件，交叉引用一个(文档化的)实体及其在源文件中的定义是很有用的。如果将SOURCE_BROWSER 标记设置为 YES, Doxygen将生成这样的交叉引用。
它还可以通过将INLINE_SOURCES 设置为 YES 将源代码直接包含到文档中(这对于代码检查非常方便)。

步骤 2: 运行doxygen

要生成文档，您现在可以输入: 

doxygen <config-file>

根据您的设置，doxygen将在输出目录中创建html、 rtf、 latex、 xml、 man、 和/或 docbook目录。顾名思义，这些目录包含以HTML、 RTF、 、 XML、 Unix-Man page以及格式生成的文档。

默认的输出目录是启动doxygen 的目录。可以使用OUTPUT_DIRECTORY更改写入输出的根目录。可以使用HTML_OUTPUT、 RTF_OUTPUT、 LATEX_OUTPUT、 XML_OUTPUT、 MAN_OUTPUT 和 DOCBOOK_OUTPUT选择输出目录中的特定格式目录。配置文件的标记。如果输出目录不存在，doxygen 将尝试为您创建它(但它不会像mkdir -p那样尝试递归创建整个路径)。

HTML 输出

生成的HTML文档可以通过将HTML浏览器指向html目录中的index.html 文件来查看。 为了获得最好的结果，应该使用支持层叠样式表的浏览器(我使用Mozilla Firefox、 Google Chrome、 Safari，有时还使用IE8、 IE9和Opera来测试生成的输出)。

HTML部分的一些特性(如GENERATE_TREEVIEW或搜索引擎)需要一个支持启用动态HTML和Javascript的浏览器。

LaTeX 输出

生成的 文档必须首先由 编译器编译(我在Linux和MacOSX中使用了一个最新的teTeX发行版，在Windows中使用MikTex)。为了简化编译生成的文档的过程，doxygen 将Makefile 写入 latex 目录(在Windows平台上也生成make.bat批处理文件)。

Makefile 中的内容和目标取决于 USE_PDFLATEX的设置。如果它被禁用(设置为NO))，那么在latex目录中键入make 以及一个叫refman.dvi的dvi文件。然后可以使用xdvi查看该文件，或者通过键入make ps(这需要 dvips)将其转换为PostScript文件refman.ps。

要在一个物理页面上放置两个页面，可以使用make ps_2on1。 生成的PostScript文件可以发送到PostScript打印机。如果没有PostScript打印机，可以尝试使用ghostscript将PostScript转换为打印机能够理解的内容。

如果您安装了ghostscript解释器，也可以将其转换为PDF; 只需输入make pdf (或 make pdf_2on1)。

要获得PDF输出的最佳结果，您应该将PDF_HYPERLINKS 要获得PDF输出的最佳结果，您应该将 USE_PDFLATEX 标记设置为 YES。在这种情况下，Makefile 只包含一个直接构建refman.pdf的目标。

RTF 输出

Doxygen将RTF输出组合到一个名为refman.rtf的文件中。这个文件是为导入Microsoft Word而优化的。某些信息是使用所谓的字段编码的。要显示实际值，您需要选择all (Edit - select all)，然后切换字段(右键单击并从下拉菜单中选择该选项)。

XML 输出

XML输出由结构化的doxygen收集的信息的“转储”组成。每个组合(类/名称空间/文件/…)都有自己的XML文件，还有一个名为index.xml的索引文件。

一个名为combine.xslt的XSLT脚本，可用于将所有XML文件合并到一个文件中。

Doxygen还生成两个XML模式文件index.xsd (用于索引文件) 和 compound.xsd (用于复合文件)。这个模式文件描述了可能的元素、它们的属性以及它们的结构，也就是说，它描述了XML文件的语法，可以用于验证或指导XSLT脚本。

在addon/doxmlparser 目录中，您可以找到一个解析器库，用于以增量方式读取doxygen生成的XML输出(有关该库的接口，请参阅addon/doxmlparser/include/doxmlintf.h)

Man page 输出

生成的手册页可以使用man 程序查看。您确实需要确保man目录位于man路径中(请参阅MANPATHMANPATH环境变量)。注意，手册页格式的功能有一些限制，因此一些信息(如类图、交叉引用和公式)将丢失。

DocBook 输出

Doxygen还可以生成DocBook格式的输出。如何处理DocBook输出超出了本手册的范围。

步骤 3: 为源文件产生文档

虽然为源文件产生文档被安排在步骤3中，但是在新项目中，这当然应该归属于步骤1。这里我假设您已经有了一些代码，您希望doxygen生成一个很好的文档来描述API，可能还有内部结构和一些相关的设计文档。

如果配置文件中的EXTRACT_ALL 选项被设置为NO(默认值)，那么doxygen将只生成文档化实体的文档。那么如何记录这些呢?对于成员、类和名称空间，基本上有两种选择:

  1. 在成员、类或命名空间的声明或定义前面放置一个特殊的 文档块。对于文件、类和名称空间成员，还允许将文档直接放在成员之后。

    有关特殊文档块的更多信息，请参见特殊注释块。

  2. 在其他地方放置一个特殊的文档块(另一个文件或另一个位置)，并在文档块中放置一个结构命令。结构命令将文档块链接到可以被文档化的特定实体(例如成员、类、名称空间或文件)。

    有关结构命令的更多信息，请参阅其他地方的文档。

第一个选项的优点是您不必重复实体的名称。

只能使用第二个选项来记录文件，因为没有办法将文档块放在文件前面。当然，文件成员(函数、变量、typedefs、定义)不需要显式的结构命令;只要在前面或后面放置一个特殊的文档块就可以了。

在将特殊文档块中的文本写入HTML和/或输出文件之前，对其进行解析。

在解析过程中，执行以下步骤:

  • 标记格式由相应的HTML或特殊命令替换。
  • 执行文档中的特殊命令。有关所有命令的概述，请参见特殊命令。
  • 如果一行以一个或多个星号(*)开始，然后是一个或多个空格(可选)，然后删除所有空格和星号。
  • 所有产生的空行都被视为段落分隔符。这将避免您为了使生成的文档可读而自己放置新段落命令。
  • 为与文档化的类对应的单词创建链接(除非单词前面有%;然后该单词将不会被链接，并且%符号将被删除)。
  • 当在文本中发现某些模式时，将创建到成员的链接。有关自动链接生成如何工作的更多信息，请参见自动链接生成 一节。
  • 文档中的HTML标记被解释并转换为与输出的 等价的 。有关所有受支持的HTML标记的概述，请参见 HTML命令 一节。
