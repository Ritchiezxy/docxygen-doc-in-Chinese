已知问题

  • Doxygen不是一个真正的编译器，它只是一个词汇扫描器。这意味着它不会检测到源代码中的错误。
  • Doxygen有一个内置的预处理器，但它的工作原理与C预处理器略有不同。Doxygen假设一个头文件被正确地保护起来，防止多重包含，并且每个包含文件都是独立的(例如，它可以放在源文件的顶部，而不会引起编译器错误)。只要这是正确的(这是一个很好的设计实践)，您就不应该遇到问题。
  • 由于不可能测试所有可能的代码片段，所以很有可能某些有效的C/ C++代码没有得到适当的处理。如果你发现这样的情况，请告知我们，我们尽量提高doxygen的解析能力。试着让你发送的代码尽可能小，以帮助我缩小搜索范围。
  • 如果代码中有多个具有相同名称的类、结构或联合，则Doxygen无法正常工作。但是，它应该不会崩溃，相反，它应该忽略所有具有相同名称的类，而只保留一个。
  • 有些命令在其他命令的参数中不起作用。在一个HTML链接中(例如<a href="…">…<a>)其他命令(包括其他HTML命令)不起作用!分段命令是一个重要的例外。
  • 在某些情况下，冗余括号会使doxygen混淆。例如: 
      void f (int);
    被正确地解析为一个函数声明，但是 
      const int (a);
    也被看作是一个名为int的函数声明，因为只分析语法，不分析语义。如果可以检测到冗余的括号，如 
      int *(a[20]);
    然后doxygen将删除括号并正确解析结果。

  • 并不是文档中包含的所有代码片段中的名称都被链接所替换(例如，当使用 SOURCE_BROWSER = YES时)，并且到重载成员的链接可能指向错误的成员。这也适用于为每个函数生成的引用列表。

    在某种程度上，这是因为代码解析器目前还不够智能。我们以后会努力改进的。但是，即使有了这些改进，也不是所有内容都可以正确地链接到相应的文档，因为可能存在关于找到代码片段的上下文的模糊或缺乏信息等的问题。

  • 如果类a已经有一个名为f的成员和相同的参数列表，则不能使 \relates 或 \relatesalso 令在类a中插入非成员函数f。
  • 目前对成员特化的支持非常有限。它只在有专门的模板类的情况下工作。
  • 并不是所有的特殊命令都被正确地翻译成RTF。
  • dot的1.8.6版本(可能还有更早的版本)没有生成正确的图表文件，导致doxygen生成的图形不能正确点击。
  • 只使用PHP: Doxygen要求所有PHP语句(即代码)都包装在函数/方法中，否则可能会遇到解析问题。

如何帮助我们

doxygen的发展高度依赖于您的投入!

如果你正在使用doxygen，请让我们了解你有什么样的可以让doxygen更棒好点子。不过我觉得您有时候可能错过了现有的一些doxygen功能。如果你决定不使用它们的某些功能，也请让我们了解为什么。

如何报告bug

bug在GitHub的问题跟踪器中被跟踪。在提交一个新的bug之前, 首先在数据库中搜索 是否其他人已经提交了相同的错误。
如果你相信你发现了一个新的错误，请报告。

如果您不确定是否有问题，请首先在 用户邮件列表 中询问帮助(需要订阅)。

如果你只发送一个(模糊的)bug描述，这个报告通常不但不会为我们提供很多帮助，甚至会花费我们很多的时间来弄清楚意思。在最坏的情况下，你的错误报告甚至可能被我们完全忽略，所以希望在你的错误报告中包括以下信息:

  • 您正在使用的doxygen版本(例如1.5.3，如果您不确定，请使用 doxygen --version命令查看）。
  • 操作系统的名称和版本号(例如Ubuntu Linux 18.04 LTS)
  • 通常，发送配置文件也是一个好主意，但请在生成配置文件时使用带-s标志的doxygen，以使其保持较小(使用doxygen -s -u [configName]从现有配置文件中删除注释)。
  • 对我来说，修复bug最简单(通常也是唯一)的方法是，您可以将演示问题的小示例附加到bug报告中，这样我就可以在我的机器上重新复现它。请确保示例是有效的源代码(可能会编译)，并且示例确实和问题匹配。 如果您打算发送多个文件，请将这些文件压缩或tar到一个单独的文件中，以便于处理。请注意，在报告新bug时，只有在提交了初始bug描述之后，才有机会向其附加文件。

您可以为一个bug添加一个补丁。如果您这样做了，请在错误输入表单中使用PATCH作为关键字。

如果您有关于如何修复现有缺陷和限制的想法，请在 开发人员邮件列表(需要订阅)中讨论它们。如果您不愿意通过bug跟踪器或邮件列表发送补丁的话，也可以将补丁直接发送到 doxyg.nosp@m.en@g.nosp@m.mail..nosp@m.com。

对于补丁，请使用“diff -uN”或包含您修改的文件。如果你发送一个以上的文件，请把所有的东西用tar或zip打包，这样我就只需要保存和下载一个文件。
