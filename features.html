Doxygen
特性

  • 只需要文档发布者的很少的开销，因为使用纯文本就可以了编写。doxygen也支持Markdown，而更高级的或结构化的输出需要使用HTML标记和/或一些doxygen的特殊命令。
  • 跨平台:适用于Windows和许多Unix版本(包括Linux和MacOSX)。
  • 即使是来自未文档化的代码，也可以建立索引、组织，生成可浏览和可交叉引用的输出。
  • 为已解析的源代码生成结构化的XML输出，外部工具可以使用这些输出。
  • 支持C/ C++， Java， (Corba和Microsoft) Java, Python, VHDL, PHP IDL, c#， Fortran, TCL, Objective-C 2.0，以及某种程度上的D源代码。
  • 支持文件、名称空间、包、类、结构、联合、模板、变量、函数、类型定义、枚举和定义的文档。
  • Javadoc(1.1)、qdoc3(部分)和ECMA-334 (c#规范)兼容。
  • 提供了一个GUI前端(Doxywizard)来简化编辑选项和运行doxygen。该GUI可用于Windows、Linux和MacOSX。
  • 自动生成HTML中的类和协作关系图(作为可单击的图像映射)和(作为封装的PostScript图像)。
  • 使用Graphviz工具包的dot工具生成包括依赖关系图、协作图、调用图、目录结构图和图形类层次图。
  • 允许对模块中的实体进行分组并创建模块的层次结构。
  • 灵活的注释放置:允许将文档放在头文件(在实体声明之前)、源文件(在实体定义之前)或单独的文件中。
  • 生成一个类的所有成员(包括任何继承的成员)及其保护级别的列表。
  • 同时以联机格式(XHTML和UNIX手册页)和脱机格式( 和RTF)输出文档(如果需要，可以禁用任何一种格式)。所有格式都经过优化，便于阅读。此外，可以使用Microsoft的HTML Help Workshop(仅限Windows)从HTML输出生成压缩的HTML，也可以从 输出生成PDF。
  • 支持各种第三方帮助格式，包括HTML帮助、docset、Qt-Help和eclipse帮助。
  • 包括一个完整的C预处理器，允许正确解析条件代码片段，并允许扩展所有或部分宏定义。
  • 自动检测公共、保护和私有部分，以及Qt特定的信号和插槽部分。私有类成员的提取是可选的。
  • 自动生成对文档化的类、文件、名称空间和成员的引用。还支持全局函数、全局变量、类型定义和枚举的文档。
  • 对基/超类和继承/覆盖成员的引用将自动生成。
  • 包括一个快速的，基于排名的搜索引擎来搜索类和成员文档中的字符串或单词(基于PHP)。
  • 包括一个基于Javascript的实时搜索功能，以搜索您键入的符号(用于中小型项目)。
  • 您可以在文档中键入普通的HTML标记。Doxygen会自动将它们转换成相应的RTF和man-page版本。
  • 允许以独立于位置的方式引用为其他项目(或同一项目的另一部分)生成的文档。
  • 允许包含与文档自动交叉引用的源代码示例。
  • 还支持包含未文档化的类，这样就可以快速地了解大段代码的结构和接口，而无需查看实现细节。
  • 允许自动交叉引用(文档化的)实体及其在源代码中的定义。
  • 为了便于阅读，所有源代码片段都突出显示了语法。
  • 允许在文档中包含函数/成员/类定义。
  • 所有选项都是从易于编辑和(可选)注释的配置文件中读取的。
  • 文档和搜索引擎可以转移到另一个位置或机器，而无需重新生成文档。
  • 支持多种不同的字符编码，并在内部和生成的输出中使用UTF-8。
  • Doxygen可以生成一个布局，您可以使用它来修改每个页面的布局。
  • 有100多个可配置选项来微调输出。
  • 能够轻松处理大型项目。

虽然doxygen现在可以用于任何由doxygen支持的语言编写的项目中，但最初它是专门为使用Qt软件的Qt工具包的项目设计的。我试着让doxygen与qt兼容。也就是说:Doxygen可以读取Qt源代码中包含的文档，并创建一个与Qt软件生成的类浏览器非常相似的类浏览器。Doxygen理解Qt使用的C++扩展，比如信号和插槽，以及Qt源代码中使用的许多标记命令。         

Doxygen还可以自动生成到现有文档的链接，这些文档是用Doxygen或Qt的非公共类浏览器生成器生成的。
对于基于Qt的项目，这意味着每当您引用属于Qt工具包的成员或类时，都会生成到Qt文档的链接。
这与文档的位置无关!
